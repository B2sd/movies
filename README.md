# Киноархив Карана

Готовая первая версия личного киноархива на **React + TypeScript + Vite**. Проект рассчитан на хостинг через **GitHub Pages** и подготовлен к подключению **Supabase** для базы, комментариев, оценок и закрытой админки.

## Что уже сделано

- Красивый темный дизайн с постерами, топом и каталогом.
- Поиск по названию, оригинальному названию, жанрам, описанию и году.
- Фильтры по типу: фильмы, сериалы, мультфильмы, аниме, шоу.
- Сортировка по дате добавления, моей оценке, году, оценке гостей и алфавиту.
- Страница фильма в модальном окне.
- Моя оценка и рецензия.
- Оценки гостей.
- Комментарии с модерацией.
- Админ-панель с демо-входом.
- Добавление новых карточек вручную.
- Сохранение демо-данных в localStorage.
- SQL-схема Supabase с RLS-политиками.
- GitHub Actions workflow для деплоя на GitHub Pages.

## Демо-админка

На главной нажми **Админка**.

Демо-код по умолчанию:

```text
karan
```

Его можно поменять в `.env`:

```text
VITE_ADMIN_DEMO_CODE=твой_код
```

Важно: демо-код в фронтенде не является настоящей защитой. Для настоящего публичного сайта нужно включить Supabase Auth и RLS, схема уже лежит в `supabase/schema.sql`.

## Подключение Supabase

1. Создай проект на Supabase.
2. Открой SQL Editor.
3. Выполни содержимое файла:

```text
supabase/schema.sql
```

4. Создай файл `.env` на основе `.env.example`:

```text
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=твой_anon_key
VITE_ADMIN_EMAIL=твой@email.com
VITE_ADMIN_DEMO_CODE=karan
```

5. В таблицу `profiles` добавь свой профиль с ролью `admin` после первого входа через Supabase Auth.

## Деплой на GitHub Pages

В репозитории GitHub:

1. Открой Settings.
2. Перейди в Pages.
3. В Source выбери **GitHub Actions**.
4. Запушь проект в ветку `main`.
5. Workflow `.github/workflows/deploy.yml` соберет и выложит сайт.

